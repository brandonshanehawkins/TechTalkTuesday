What are Agents?

AI-powered assistants tailored to specific purposes with defined capabilities and limitations.

  • Automate tasks, provide insights, and enhance decision‑making.
  • Types: Project, Workspace, and Certified Agents.

Why create Agents?

  • Reduce repetitive work and errors; improve speed and consistency.
  • Adapt quickly via feedback; optimize workflows and costs.
  • Improve decisions with data; personalize using grounded knowledge.

7‑Step build

1Prepare Data — clean, structured, concise; prefer .txt.
2Upload Data — add relevant docs in the Data tab.
3Define Profile — name, description, Expert/Persona, image, tone.
4Instructions — system prompt, tools, optional CoT/Steps.
5Test — Evaluation + UAT.
6Publish — start at project level.
7Iterate — refine via feedback, memories, testing.

Optimizing data

Golden rules
  • Keep language clear and formatting consistent.
  • Avoid huge files (> 50 MB) and cluttered documents.
By file type
  • Word/PDF: extract text; use headings and lists; OCR if needed.
  • PowerPoint: pull slide text; keep order; remove decorative visuals.
  • Excel: tidy columns; avoid merged cells; keep tables small (~≤100 rows).
  • Images: provide concise descriptions.
  • Audio/Video: add transcripts and time‑coded summaries.

Agent profile

  • Expert: task specialist and knowledge hub.
  • Persona: audience POV and preferences.
  • Set personality/tone and image; choose tools and capabilities.

Advanced settings

  • Temperature: creativity (0.2 factual → 1.5+ creative).
  • Top P: focus (lower = deterministic).
  • Max Tokens: response length.
  • Top K: word‑choice variety. Tip: Defaults work best.

Instructions & tools

System prompt, CoT, and Steps
  • System Prompt sets role, scope, rules, and formatting.
  • Chain of Thought for reasoning‑heavy tasks (don’t mix with Steps).
  • Steps (Pro) define multi‑stage flows with per‑step actions.
  • Required Info + Input Form (beta) collect structured inputs.
Tools
  • Web Search — real‑time info.
  • Code Interpreter — Python for data/automation.
  • Artifact Generation — produce docs and structured outputs.

Testing

  • Evaluation (Automated): inputs + ideal outputs → quality score; fix failures.
  • UAT: real users verify clarity, tone, and edge cases.
  • Re‑test after changes; keep scenarios in a spreadsheet for reuse.

Feedback & learning

  • Feedback can turn into Memories that shape responses.
  • Guardrails enforce boundaries; User memories personalize.
  • Learning Mode (beta) captures improvements automatically.

Publishing & sharing

  • Publish first at project level; expand after testing.
  • Share to brand, client, market, or tenant (permissions required).
  • Agents appear in Chat, AI Tools, Canvas, Content Matrix (Personas), Focus Groups.

Managing agents

  • Set Default for reliability across workflows.
  • Edit, Duplicate, Delete; keep a Template agent for variations.

Best practices

  • Define purpose and target audience/tone.
  • Clean, minimal data; keep prompts concise; avoid redundancy.
  • Test broadly; review feedback; iterate regularly.

Agentic Team

Beta model that coordinates multiple Agents in a project to work together on a goal.

Agents vs Brains vs AI Assistant

  • AI Assistant orchestrates workflows and invokes agents/brains.
  • Agents perform grounded tasks using your uploaded knowledge.
  • Brains are deeply trained services for complex, brand‑specific outputs.

Examples

  • Brand/Product Experts; Playbooks & Guidelines.
  • Persona/Audience; Channel Format Experts.
  • Project Brief Agents; Retail/Category Experts.
